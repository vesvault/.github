# Contributing to VESvault

Thanks for your interest in improving VES! This guide is the default for all
repositories in the **VESvault** organization; an individual repo may add its
own `CONTRIBUTING.md` with project-specific details that take precedence.

## Ways to contribute

- **Ask a question or share an idea** →
  [Discussions](https://github.com/orgs/vesvault/discussions)
- **Report a bug** → open an Issue in the relevant repository with steps to
  reproduce, expected vs. actual behavior, and your environment (OS, version or
  commit).
- **Report a security vulnerability** → please **do not** open a public issue —
  follow our [Security Policy](https://github.com/vesvault/.github/security/policy).
- **Improve code or docs** → open a pull request (see below).

## Pull requests

1. Fork the repository and create a topic branch from the default branch.
2. Keep each PR focused, and match the existing style of the file you're editing.
3. Build the project and run its tests — see the repository's `README` for how.
4. Write a clear description: **what** changed, **why**, and **how you verified** it.
5. Link any related issue or discussion.

Small fixes (typos, docs, obvious bugs) are welcome without prior discussion.
For larger or design-affecting changes, please open a Discussion or Issue first
so we can agree on the approach before you invest the work.

## Licensing

The VES client libraries and the `ves` CLI are released under **Apache-2.0**. By
submitting a contribution, you agree that it is licensed under the same license
as the repository you are contributing to.

## Code of Conduct

Participation in VESvault projects is governed by our
[Code of Conduct](https://github.com/vesvault/.github/blob/main/CODE_OF_CONDUCT.md).
By participating, you are expected to uphold it.
